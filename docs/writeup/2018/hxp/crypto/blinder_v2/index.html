crypto - blinder_v2

blinder_v2

Please hack.

Time

A day

Behavior

Same as blinder, but I found a more efficient solution for this particular ppp. It takes about 4500 operations to work (c.f. 13000 operations using elliptic curve).

The previous solution using elliptic curve is more general. This solution won't work if you select a prime with non-smooth factorization for (pd−1)/(p−1)(p^d-1) / (p-1)(pd−1)/(p−1).

Solution

TL;DR

  1. Work under the polynomial quotient ring of modulo x2+1x^2 + 1x2+1.
  2. Recover the partial index of yyy using Pohlig-hellman.
  3. Calculate yyy.

Note: If you haven't read our previous solution using elliptic curve, read it first. The concepts are similar, and I also applied most of the optimizations here.

After the previous solution is finished, I try to figure out how elliptic curve punch some holes in that field, creating a shrinking map to a smaller subgroup which has a smooth order.

Some points not on the curve seems to be the reason, but I still don't know how these things actually works, so let's skip this part.

Elliptic curve works really well on cryptography due to the hardness of its DLP. But clearly, we don't need such thing in this task. Maybe we can work on another simpler structure than elliptic curve.

Here it is: Polynomial quotient ring !!

Polynomial Quotient Ring

Let fff to be an irreducible polynomial over Fp\mathbb{F}_pFp​. The order of multiplicative group in the quotient ring (Fp[x]/⟨f⟩)∗(\mathbb{F}_p[x]/\langle f \rangle)^*(Fp​[x]/⟨f⟩)∗ is p2−1p^2 - 1p2−1. In this task, the order is actually 24×3×5×59×281×3037×23293×(p−1)2^4 \times 3 \times 5 \times 59 \times 281 \times 3037 \times 23293 \times (p-1)24×3×5×59×281×3037×23293×(p−1) , which is almost smooth except for the last factor.

Recover part of the index

It won't be too hard to recover the index of a polynomial in each subgroup except last one. 303730373037 and 232932329323293 is a little bit large, so we need to use baby step giant step here. Luckliy, multiplication of polynomial doesn't cost too much operations.

Reconstruct y

Clearly, there's no efficient way to recover the index in (p−1)(p-1)(p−1) subgroup using oracle only. But we don't need to know that, we already have enough information to reveal what yyy is.

Let's think about what that (p−1)(p-1)(p−1) subgroup is?

It's a group of all constants (i.e. degree 0 polynomials).
